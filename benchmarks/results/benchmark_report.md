### 关键发现

1. **pywin32 整体更稳定**: Python 的动态类型天然适配 COM 的 late-binding，几乎不需要类型转换
2. **csharp 批量写入更快**: C# 的 `object[,]` 2D 数组直接映射 COM SAFEARRAY，在大批量写入场景有优势
3. **csharp 编译开销**: CSharpCodeProvider 的编译时类型检查是双刃剑——捕获了真正的错误，但也增加了开发迭代成本
4. **逐行 COM 调用是瓶颈**: P3/P6 中逐行操作的延迟两个后端都高，应优先使用 Range 批量操作
5. **多工作簿管理**: C# server 的工作簿状态管理不如 pywin32 清晰（excel_open 没有正确切换活动工作簿）

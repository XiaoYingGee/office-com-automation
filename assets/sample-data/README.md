# Sample Data · 共享样例数据

> 所有语言、所有基准都用**同一份可复现数据**，保证 PK 公平、结果可重跑。
> 本期约定生成规则；具体数据/生成脚本在 P1 阶段补。

## 原则

- **可复现**：数据由固定 `seed` 的伪随机生成，任何语言/任何时间生成结果一致。
- **统一规模**：与 [benchmark-spec](../../docs/benchmark-spec.md) 的 S/M/L 档对齐。
- **混合类型**：含字符串、整数、浮点、日期，覆盖 `Value2` 的类型场景与 `NumberFormat`。

## 数据规格（约定）

每行结构（M 档 10 列示例）：

| 列 | 名称 | 类型 | 说明 |
|---|---|---|---|
| A | id | 整数 | 行号/主键 |
| B | name | 字符串 | `item_<n>` |
| C | category | 字符串 | 有限枚举（用于 E08 排序/筛选） |
| D | qty | 整数 | 0–1000 |
| E | price | 浮点 | 两位小数（用于 NumberFormat/公式） |
| F | amount | 公式位 | `=D*E`（E04 用） |
| G | date | 日期 | 用于 Value vs Value2 对照 |
| H–J | 备用数值 | 浮点 | 填充到目标列宽 |

> L 档扩展到 20 列，多出的列重复 H–J 的数值模式。

## 生成约定（P1 落地）

- 提供一个**语言中立**的生成说明（或单一参考脚本），各语言据此本地生成，避免把大文件入库。
- 固定 `seed`（写进 `benchmark-spec` 的环境元数据）。
- 生成产物（`.xlsx`/`.csv`）默认 git 忽略；仅保留生成规则与一个**小样例**（如 S 档前 20 行）供肉眼核对。

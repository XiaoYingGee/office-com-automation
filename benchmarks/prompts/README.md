# MCP Benchmark Prompts

> 用这些 prompt 分别测试 excel-pywin32 / excel-addin / excel-csharp 三个 MCP server。
> 每个 prompt 测量：响应延迟、正确性、错误恢复能力。
>
> 测试时请记录：使用哪个 MCP server、文件大小、总耗时、是否成功。

---

## P1: 基础读写（empty）

```
打开 benchmarks/fixtures/bench_empty.xlsx，在 Sheet1 的 A1:E5 区域写入一个 5x5 的九九乘法表
（A1=1x1=1, B1=1x2=2, ... E5=5x5=25），然后把第一行加粗，保存。
```

**考察：** 单次调用写入 + 格式化

---

## P2: 批量数据写入（empty）

```
打开 benchmarks/fixtures/bench_empty.xlsx，在 Sheet1 写入 1000 行 x 10 列的数据：
- A列：行号（1-1000）
- B列：随机部门名（销售/研发/市场/财务）
- C列：随机金额（1000-99999，保留2位小数）
- D-J列：该行金额的各种计算（×2, ×3, 累计, 平均, 最大, 最小, 方差）

写完后设置 A1:J1 为表头（加粗、蓝色背景），数字列设置千分位格式。保存。
```

**考察：** 循环写入 + 计算 + 格式化（CodeAct 模式优势场景）

## P3: 数据查询与条件格式（medium）

```

**考察：** 大文件打开 + 遍历 + 条件格式 + 返回统计

## P4: 跨表汇总（medium）

```
打开 benchmarks/fixtures/bench_medium.xlsx。
新建一个"汇总"工作表，统计"销售数据"表中每个部门的总金额和总数量：
- A列：部门名
- B列：总金额（SUM）
- C列：总数量（SUM）
- D列：平均单价（总金额/总数量）

按总金额降序排列，保存。
```

**考察：** 多表操作 + 数据聚合 + 排序

## P6: 查找替换 + 公式（medium）

```

1. 把所有"产品A"替换为"旗舰产品"

## P8: 错误恢复（empty）

```
打开 benchmarks/fixtures/bench_empty.xlsx。
尝试以下操作（其中有些会失败，请处理错误继续执行）：
1. 在 A1 写入 "test"
2. 尝试删除不存在的工作表 "NotExist"（预期失败）
3. 在 A2 写入 100
4. 尝试读取不存在的单元格 "ZZZ999999"（预期失败或返回空）
5. 在 A3 写入当前时间字符串

返回每步的执行结果（成功/失败及原因）。
```

**考察：** 错误处理 + 自愈能力 + 结构化反馈

## P9: 复杂格式化（empty）

```
打开 benchmarks/fixtures/bench_empty.xlsx，创建一个"月度报表"模板：
1. 合并 A1:F1，写入"2024年度销售月报"，字号20，加粗，居中
2. A2:F2 写入表头：月份、目标、实际、完成率、差额、评级
3. A3:A14 写入 1月-12月
4. B3:B14 写入目标值（100000）
5. C3:C14 写入随机实际值（80000-120000）
6. D3:D14 写入完成率公式 =C/B
7. E3:E14 写入差额公式 =C-B
8. F3:F14 写入评级公式 =IF(D>=1,"达标","未达标")
9. 设置 D列为百分比格式，B/C/E列为千分位格式
10. 为完成率列添加条件格式：>=100% 绿色背景，<100% 红色背景

保存。
```

**考察：** 合并单元格 + 公式 + 条件格式 + 多步骤协调

| 维度 | 权重 | 说明 |
|------|:---:|------|
| 正确性 | 40% | 操作结果是否正确 |

| 错误恢复 | 15% | 遇到错误时能否自愈 |
| 代码质量 | 15% | 生成的代码是否简洁、高效（CodeAct 模式） |
